History

In the 1970s when Computer Science was experience its Big Bang moment, scientists started formalizing algorithms to solve the world problems. While doing that they realized that it was significantly easier to come up with a fast algorithm for some problems (sorting an array) as compared to some other problems (Traveling Salesman Problem). This raised an interesting question, "Are some problems significantly difficult to solve as compared to other problems?". The answer is not as obvious as it seems!

For example, problems like Discrete Fourier Tranform or Finding Prime Numbers were considered 'slow' earlier but now we have 'fast' algorithms to solve them. To formalize this, scientists started categorizing all the problems into two major categories P and NP.

P (Polynomial)

The category P contains all those problems which can be solved in polynomial time. Here we are considering worst case scenario while talking about the time complexity. Problems like sorting an array, multiplying two numbers, detecting cycle in a graph fall into this category.

NP (Non-deterministic Polynomial)

The category NP contains all those problems where, given a solution, checking whether the solution is correct or not can be done in polynomial time. Problems like soving a Sudoku, Graph coloring, SAT fall into this category. There are no polynomial run time algorithms found for solving a sudoku but, given a solved sudoku board, it's easy to check if the solution is correct in polynomial time.

Note that a computer can solve a 9x9 sudoku board in milliseconds, but complexity comes into play when we scale the problem. A computer will have no problem sorting an array of a billion integers, but will take significantly more time to solve a billion sized sudoku board.
